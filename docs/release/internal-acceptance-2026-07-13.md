## 三、复验结论

### 已关闭：上海运营日 UTC 日期统计错误

- 修复方式：订单和车辆任务时间均先转换至 `Asia/Shanghai`，再取运营日日期；接口默认日期也使用上海时区。
- 自动化验证：覆盖 `2026-07-12T23:17:00Z` 归属 `2026-07-13` 的订单和任务统计场景。
- 真实复验：看板显示与上海运营日一致，原问题关闭。

### 已关闭：车辆任务页选择上下文与执行鉴权

- 复验对象：订单 `724d0e0d`，车辆任务 `9081cbff`，登录账号 `dispatcher01`。
- 根因：初验时 `5173` 提供的是旧工作树的前端代码，而 `8082` 提供的是当前工作区 API，前后端版本不一致导致页面任务上下文和访问令牌行为混杂。
- 修复与验证：统一为当前工作区的前端和 API 后，页面可选中非首条待执行任务；新增页面回归测试覆盖该场景；真实浏览器已完整跑通车辆任务状态机。
