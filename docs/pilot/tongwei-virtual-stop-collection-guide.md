# 通渭县虚拟站点采集与导入指南

## 采集目标

首批整理 30-50 个高频上下车点，优先覆盖公交站点、小区出入口、医院、学校、政务服务点和商圈。每条记录必须明确实际上/下车的安全位置，而非仅使用建筑中心点。

## 字段口径

使用 [导入模板](tongwei-virtual-stops-template.csv)，逐行填写：

| 字段 | 要求 |
| --- | --- |
| 站点名称 | 清晰、唯一，例如“通渭县人民医院北门” |
| 地址 | 可被调度员识别的完整地址或地标 |
| 经度、纬度 | GCJ-02 坐标，保留 6 位小数 |
| 所属区域 | 已发布服务区名称 |
| 服务半径 | 常用 100-500 米，避免覆盖不安全道路 |
| 允许上车、允许下车 | 填写是/否，按道路和站点条件决定 |
| 安全说明 | 路侧、候车区、禁停和施工等注意事项 |

## 采集与校验流程

1. 调度员收集候选点，记录来源和联系人。
2. 运营人员在高德地图核对地址和 GCJ-02 坐标；必要时现场确认。
3. 用 CSV 导入预校验，处理重复名称、无效坐标和服务区外提示。
4. 在虚拟站点地图抽查所有医院、学校、小区、公交站等高频点及至少 20% 的其他点。
5. 指定管理员启用通过核验的站点；问题站点保留为停用并修正，不删除原始采集结论。
6. 导出导入结果，保留采集表、截图和修正记录，供试点复盘。

## 质量规则

- 不以导航道路中线替代安全候车点。
- 两个站点相距很近但服务方向不同，应保留不同名称和安全说明。
- 坐标或地址不确定时保持停用，不能为凑数量启用。
- 任何服务区外站点只可作为待修正记录，不能进入试点派单候选。
