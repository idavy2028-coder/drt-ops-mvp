# 通渭县地图与虚拟站点试点准备清单

## 试点范围

- 区域：甘肃省通渭县；服务区由运营人员在后台划定并发布。
- 周期：1 个月；运营时段：06:30-19:00。
- 人员与车辆：4 辆车、4 名驾驶员、2 名调度员。
- 订单来源：调度员根据电话或微信手工录入；不接入乘客端、司机端或车载 GPS。
- 位置口径：驾驶员电话/微信反馈节点位置，调度员录入。地图显示为“人工上报”，不是连续轨迹。

## 上线前配置

1. 在高德控制台分别创建 Web 服务和 JS API 应用；限制 JS Key 的试点域名/本机开发域名，并设置安全密钥。
2. 仅在受控环境注入以下变量，禁止写入代码、`.env` 模板或 Git：

```powershell
$env:DRT_AMAP_ENABLED = "true"
$env:DRT_AMAP_WEB_SERVICE_KEY = "实际 Web 服务 Key"
$env:VITE_AMAP_ENABLED = "true"
$env:VITE_AMAP_JS_API_KEY = "实际 JS API Key"
$env:VITE_AMAP_SECURITY_JS_CODE = "实际安全密钥"
```

3. 确认 PostgreSQL/PostGIS 已部署、每日备份已启用，并为 API、算法服务和管理端分别配置健康检查。
4. 使用管理员账号发布服务区边界；由公交企业确认边界版本、启用状态和 GCJ-02 坐标系。
5. 导入 30-50 个候选虚拟站点后，完成现场或远程地图抽查，未核验站点不得启用。

## 运行降级

- 高德服务或 JS 地图不可用：继续用经纬度、地址和虚拟站点手工录入；不得阻断订单、任务或人工位置上报。
- 路径/ETA 不可用：调度员以电话确认和人工判断派单；不得将估算 ETA 当作准点率硬依据。
- 无 GPS：车辆位置只在发车、到站、上车、下车和完成等节点更新。
- 服务区外的车辆位置可保存并告警；订单起终点在已发布服务区外则应被拒绝。

## 发布门槛

以下项目均有记录后，方可进入试点：服务区已发布、站点采集与抽检完成、免费配额确认、完整动作链验收通过、备份恢复演练完成、监控与告警责任人明确。

## 当前未关闭风险

- 真实高德 Key、域名白名单和免费配额须由企业高德账号实际确认。
- 生产 PostgreSQL/PostGIS、约 1 万条位置事件容量和备份恢复尚需在目标环境实测。
- 人工位置上报存在反馈延迟；精细 ETA 与准点率仅作辅助观察，不作为试点初期硬性考核。
