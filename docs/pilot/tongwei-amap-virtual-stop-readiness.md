# 通渭县地图与虚拟站点试点准备清单

## 试点范围

- 区域：甘肃省通渭县；服务区由运营人员在后台划定并发布。
- 周期：1 个月；运营时段：06:30-19:00。
- 人员与车辆：4 辆车、4 名驾驶员、2 名调度员。
- 订单来源：调度员根据电话或微信手工录入；不接入乘客端、司机端或车载 GPS。
- 位置口径：驾驶员电话/微信反馈节点位置，调度员录入。地图显示为“人工上报”，不是连续轨迹。

## 上线前配置

1. 管理端默认使用 Leaflet 与 OSM 标准公开瓦片 `https://tile.openstreetmap.org/{z}/{x}/{y}.png`，页面必须可见 OpenStreetMap 归属信息。
2. 受控部署环境如需覆盖默认值，只配置 `VITE_TILE_URL`、`VITE_TILE_ATTRIBUTION` 与 `VITE_TILE_MAX_ZOOM`；不得将瓦片服务凭据写入代码、`.env` 模板或 Git。
3. 只允许交互式浏览器按需加载瓦片；禁止预取、离线下载、服务端代理和批量缓存。公共 OSM 无 SLA，不得将其可用性视为生产承诺。
4. 确认 PostgreSQL/PostGIS 已部署、每日备份已启用，并为 API、算法服务和管理端分别配置健康检查。
5. 使用管理员账号发布服务区边界；由公交企业确认边界版本、启用状态和 GCJ-02 坐标系。
6. 导入 30-50 个候选虚拟站点后，完成现场或远程地图抽查，未核验站点不得启用。

## 运行降级

- 开放底图不可用：继续用经纬度、地址和虚拟站点手工录入；不得阻断订单、任务或人工位置上报。
- 路径/ETA 不可用：调度员以电话确认和人工判断派单；不得将估算 ETA 当作准点率硬依据。
- 无 GPS：车辆位置只在发车、到站、上车、下车和完成等节点更新。
- 服务区外的车辆位置可保存并告警；订单起终点在已发布服务区外则应被拒绝。

## 发布门槛

以下项目均有记录后，方可进入试点：服务区已发布、站点采集与抽检完成、开放瓦片合规核对、完整动作链验收通过、备份恢复演练完成、监控与告警责任人明确。

## 当前未关闭风险

- 公共 OSM 瓦片无 SLA，网络受限或服务受限时仅保证文本、站点和手工坐标降级流程；后续可切换已获授权的瓦片服务或官方高德增强能力。
- 生产 PostgreSQL/PostGIS、约 1 万条位置事件容量和备份恢复尚需在目标环境实测。
- 人工位置上报存在反馈延迟；精细 ETA 与准点率仅作辅助观察，不作为试点初期硬性考核。
